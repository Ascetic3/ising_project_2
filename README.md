## Модель Изинга на Go: консольный движок

Репозиторий содержит реализацию двумерной модели Изинга на решётке \(L \times L\) с методом Монте‑Карло (алгоритм Метрополиса).  
Программа работает как консольный «движок»: читает параметры эксперимента из `input.csv` и записывает усреднённые наблюдаемые в `results.csv`.

## Формат входного файла `input.csv`

Разделитель — точка с запятой `;`.  
Первая строка может быть заголовком и будет пропущена, если в первом поле стоит `L`.

Каждая последующая строка имеет вид:

```text
L;J1;J2;J3;J4;J5;J6;copies;h;T;aSteps;mSteps;save
```

Где:

- `L` — размер решётки \(L \times L\);
- `J1…J6` — обменные интегралы (сейчас в модели используется только `J1`, остальные просто проходят через ввод/вывод);
- `copies` — число независимых копий решётки;
- `h` — внешнее поле;
- `T` — температура;
- `aSteps` — число шагов термализации;
- `mSteps` — число шагов измерения;
- `save` — режим использования состояния решётки:
  - `0` — перед расчётом строки состояние обнуляется и задаётся ферромагнитная конфигурация (все спины `+1`);
  - `!=0` — расчёт продолжается с конфигурации, полученной на предыдущей строке для тех же `(L, copies)`.

Важно: все поля читаются как есть и в таком же порядке попадают в выходной файл.

## Формат выходного файла `results.csv`

Разделитель — точка с запятой `;`.

Каждая строка `results.csv` соответствует одной строке из `input.csv` и имеет вид:

```text
L;J1;J2;J3;J4;J5;J6;copies;h;T;aSteps;mSteps;save;E;E2;M;M2;Afm;Afm2
```

Где:

- первые 13 полей — точная копия входной строки (`input-params`);
- `E` — средняя энергия;
- `E2` — средний квадрат энергии;
- `M` — средний модуль полного магнитного момента;
- `M2` — средний квадрат полного магнитного момента;
- `Afm` — средний модуль антиферромагнитного порядка;
- `Afm2` — средний квадрат антиферромагнитного порядка.

Наблюдаемые усредняются по измерительным шагам и по копиям.

## Как запускается моделирование

Основная логика реализована в двух файлах:

- `ising/ising.go` — тип `Simulator` и функции для шага Метрополиса и расчёта наблюдаемых;
- `cmd/run/main.go` — код, который:
  - читает `input.csv`,
  - управляет созданием/сбросом симулятора с учётом `save`,
  - вызывает моделирование для каждой строки,
  - пишет результаты в `results.csv`.

### Симулятор

Тип `Simulator` в `ising/ising.go` хранит набор решёток:

- `NewSimulator(L, copies)` создаёт симулятор и инициализирует его ферромагнитным состоянием (все спины `+1`);
- `ResetFerromagnetic()` переинициализирует решётки тем же состоянием;
- `Run(J, h, T, aSteps, mSteps)`:
  - выполняет термализацию (`aSteps`) и измерения (`mSteps`) при температуре `T`;
  - обновляет конфигурацию решёток «на месте»;
  - возвращает одну структуру `ResultRow` с полями `T, U, M, C, X, E, E2, Mtot, M2, Afm, Afm2`.

### Логика `save` в `cmd/run/main.go`

При чтении каждой строки:

- если это первая строка для данного `(L, copies)` или `save == 0`, создаётся новый `Simulator(L, copies)` (ферромагнитное состояние);
- если `save != 0` и `(L, copies)` совпадают с предыдущими, используется уже существующий симулятор, и расчёт продолжается с последнего состояния.

## Запуск

Из корня репозитория:

```bash
go run ./cmd/run
```



До запуска нужно подготовить `input.csv` в описанном формате; после завершения работы появится `results.csv` с тем же числом строк (без учёта заголовка), где к каждому набору параметров добавлены наблюдаемые `E, E2, M, M2, Afm, Afm2`.
